# Requirement design documents

One document per Requirement issue that needs a long-form design, named
`{issue_number}_design.html` (e.g. `57_design.html` for issue #57) and linked from the
issue's Design section. Simple requirements keep their design inline in the issue
body instead. See the [Requirement lifecycle](../../README.md#requirement-lifecycle).

Design documents are **self-contained HTML pages** (site palette, no external build
step) so they render directly on the hub at
`https://obot-claw.github.io/requirements/design/{issue_number}_design.html`.
Each carries a status line (Draft / Signed off) and an attribution line; sign-off
happens on the Requirement issue before sub-issues are filed.
