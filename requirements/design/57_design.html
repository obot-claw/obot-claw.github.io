Context and goals

#57 consolidates the nine legacy safetyGraphics renderers into one modern Chart.js library at jwildfire/safety.viz, consumed by Widget_*.R htmlwidgets in gsm.safety — mirroring how gsm.kri consumes gsm.viz. This document settles the repo layout, module contract, stack, traceability approach, and the distribution decision.

Non-goals: no shared-utility layer until the second renderer forces one; no npm publication in the first iteration; no visual-regression or accessibility hardening in the pilot (tracked as explicit follow-ups).

Module contract

Every renderer module exposes the lifecycle API proven by the histogram pilot as its public surface — this is what widget bindings and test harnesses call:

MethodBehavior
init(el, data, settings)Mount into a container element; validate inputs (checkInputs) and render.
setData(data) / setSettings(settings)Replace inputs; re-derive config and datasets.
render() / resize()Redraw; respond to container size changes.
destroy()Tear down the Chart.js instance and listeners; container returns to pre-init state.

Internally each module follows the gsm.viz flow (checkInputs → configure → structureData → getPlugins/getScales → new Chart(...)), so the file layout stays recognizably parallel to the reference implementation even though the public API is the pilot's lifecycle shape.

Traceability and evidence

  • Test names are keyed to requirement IDs from the safety-agent matrices, e.g. test('SH-CTRL-004: normal range checkbox displays overlay when available', …).
  • The matrix Evidence Type column routes each row: unit rows → Vitest; browser rows → Playwright.
  • Each module maintains a coverage table (requirement ID → test file) in docs/, the qcthat-style traceability artifact on the JS side.
  • Development follows the red-green TDD skill: matrix rows → failing tests → minimal implementation.

Distribution — the decision this document settles

Recommendation: vendored, versioned dist/ directory; defer npm.

  • esbuild emits dist/safety.viz-{version}/safety.viz.js (IIFE/UMD for widgets) and an ESM build; the versioned directory is committed to the repo.
  • gsm.safety vendors it at inst/htmlwidgets/lib/safety.viz-{version}/ — the exact precedent set by gsm.kri, which ships inst/htmlwidgets/lib/gsm.viz-2.3.0/.
  • R packages must build offline from a plain clone (GxP-friendly, no node toolchain at install time), which vendoring guarantees.
  • npm publication is deferred until the module API has survived at least renderer #2 (shift plot) without breaking changes; nothing in this layout blocks publishing later.

CI

One GitHub Actions workflow on PRs and main: npm ci → build → Vitest → Playwright (with browser install step). A drift check fails the build if dist/ doesn't match a fresh build of src/.

Dependencies and sequencing

  • First module is the histogram, extracted from safety-histogram dev (merged PR #1) under Requirement #58 slice ② — see 58_design.html.
  • Renderer order thereafter: shift plot (39 matrix rows) → the remaining seven.
  • The repo lives under jwildfire for now and may transfer to obot-claw; nothing in this design depends on the owner.

Open questions

  • Examples site scope — defer building a GitHub Pages examples gallery until 2–3 modules exist; the Playwright test pages double as demos meanwhile.
